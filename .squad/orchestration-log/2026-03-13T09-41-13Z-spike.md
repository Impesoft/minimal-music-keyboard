## Summary

Identified root cause of OB-Xd editor hang: thread affinity mismatch between bridge's synchronous pipe-read loop (main thread) and JUCE's `MessageManager` singleton (also main thread). Deadlock occurred when `attached()` tried to post a callback to the message manager thread.

Decision: Move pipe reading to background thread; main thread now runs `GetMessageW` message loop. All VST3 operations (load, editor open/close) now happen on the UI/STA thread with active message pump.

## Architectural Invariant Documented

> **The bridge's main thread is the UI/STA thread.** All window creation, COM operations, plugin loads, and IPlugView calls must happen on this thread. The pipe reader is a background thread that only enqueues strings and posts window messages.
