## Blocking Issues (Fixed Pre-Commit)

### Bug 1: C++ HostApplication Lifetime
- **Severity:** High (Dangling Pointer)
- **Location:** `audio_renderer.cpp` `Load()` method
- **Problem:** HostApplication created as local `IPtr`; destroyed when `Load()` returns; plugins store raw pointer
- **Fix Applied:** Added `Steinberg::IPtr<HostApplication> hostApp_;` member; store in `Load()`, release in `ResetPluginState()`
- **Status:** ✅ FIXED

### Bug 2: C++ IEditController Teardown
- **Severity:** Medium (Use-After-Free)
- **Location:** `audio_renderer.cpp` `ResetPluginState()` method
- **Problem:** `terminate()` called without first `disconnect()` on IConnectionPoint
- **Fix Applied:** Added disconnect logic before terminate calls
  - Query `IConnectionPoint` from both component and controller
  - Call `disconnect()` if supported
  - Only then call `terminate()`
- **Status:** ✅ FIXED

## Non-Blocking Notes (Phase 2)

### Note 1: C# _lastReadPos Thread Safety
- **Severity:** Low (Audio Glitches)
- **Location:** `Vst3BridgeBackend.cs` `Read()` method
- **Issue:** TOCTOU race between `writePos` read and audio buffer copy
- **Mitigation:** Race window small (~microseconds copy time vs 20ms frame); acceptable for MVP
- **Future Fix:** Double-buffering or Seqlock pattern in Phase 2
- **Status:** ✅ ACCEPTED (MVP scope)
