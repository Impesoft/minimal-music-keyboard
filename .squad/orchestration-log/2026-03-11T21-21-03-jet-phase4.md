### Design Decisions
1. **Slot-based Architecture** — Each VST3 slot gets unique `InstrumentDefinition` (consistent with SF2 catalog pattern)
2. **Catalog Persistence** — VST3 configs stored in `instruments.json`, survive app restarts
3. **Backward Compatibility** — SF2 selection unchanged; SF2 default (index 0), existing workflows untouched
4. **No Breaking Changes** — All existing instrument selection continues to work
